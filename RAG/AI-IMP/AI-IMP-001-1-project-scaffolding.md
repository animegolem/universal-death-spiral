# AI-IMP-001-1-project-scaffolding

## Initialize Go Module and Project Skeleton

The repository currently contains only documentation (README, LICENSE, RAG directory). This ticket creates the Go module, directory structure, configuration loading, and entry point that all subsequent work builds on.

**Done when:** `go build .` compiles a binary that loads a `.env` file and prints the loaded configuration, then exits. The directory structure is in place for the remaining IMP tickets to populate.

### Out of Scope

- API client code (IMP-001-2)
- Tool implementations (IMP-001-3)
- Game loop logic (IMP-001-4)
- Tests beyond verifying the build compiles and config loads

### Design/Approach

**Go module path:** `github.com/animegolem/universal-death-spiral` — matches the GitHub repo. This is the import path other packages within the module use to reference each other.

**Directory structure:**
```
main.go                    # Entry point — wires components, starts the loop
internal/
├── api/                   # Provider interface + implementations (IMP-001-2)
├── tools/                 # Tool registry + individual tools (IMP-001-3)
├── game/                  # Game loop (IMP-001-4)
└── config/                # Config loading (this IMP)
```

**Why `internal/`?** Go's `internal` directory is special — the compiler enforces that code inside it can only be imported by packages rooted at the parent of `internal`. In plain terms: our code can use `internal/config`, but nobody who imports our module from outside can. This gives us freedom to change internal APIs without worrying about breaking external consumers. It's standard Go practice for application code.

**Config loading:** We use `joho/godotenv` to load `.env` files. This is a tiny library (zero transitive dependencies) that reads key=value pairs from a file and sets them as environment variables. The config package then reads those env vars into a typed struct.

**Config struct:**
```go
type Config struct {
    APIKey    string // OPENROUTER_API_KEY
    Model     string // MODEL (default: "anthropic/claude-sonnet-4-20250514")
    MaxTokens int    // MAX_TOKENS (default: 4096)
}
```

**Dependencies:**
- `joho/godotenv` — `.env` file loading
- `sashabaranov/go-openai` — added here so `go.sum` is complete, used in IMP-001-2

### Acceptance Criteria

**Scenario:** Developer clones the repo and builds for the first time.
**GIVEN** Go 1.21+ is installed and the repo is cloned.
**WHEN** The developer runs `go build .` in the project root.
**THEN** A binary is produced without errors.
**AND** `go vet ./...` reports no issues.

**Scenario:** Developer runs the binary with a `.env` file.
**GIVEN** A `.env` file exists with `OPENROUTER_API_KEY=sk-or-test123`.
**WHEN** The developer runs the binary.
**THEN** The binary prints the loaded configuration (model, max tokens) with the API key redacted.

**Scenario:** Developer runs the binary without an API key.
**GIVEN** No `.env` file exists and `OPENROUTER_API_KEY` is not set.
**WHEN** The developer runs the binary.
**THEN** The binary exits with a clear error message indicating the API key is missing.
